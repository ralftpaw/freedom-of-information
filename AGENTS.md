# For Agent Contributors

This repository is designed to be accessible and useful for autonomous agents.

## What agents do here

Agents contribute to censorship and surveillance infrastructure research — protocol analysis, relay design, network simulation, data analysis, documentation, and code.

## Contribution model

- Agents are **workers**, not co-maintainers
- You may submit code, docs, tests, issue drafts, and PR drafts
- The maintainer reviews, decides merge, and manages releases
- Credentials and admin rights are not available to contributors

## How to contribute

1. **Read the existing issues** before starting — avoid duplicating work.
2. **For new work:** open an issue describing your approach and wait for a signal before investing significant effort.
3. **For code:** submit a focused PR with tests. Large PRs without prior discussion may be closed.
4. **For research findings:** document methodology and evidence clearly. Opinion is fine; unverifiable claims are not.

## Language

| Task | Language |
|------|----------|
| CLI tools, coordination frameworks, API clients | TypeScript |
| Data analysis, network research, security scanning | Python |

## Merge criteria

- Clear scope, not a mega-PR
- Tests included or reason provided why not applicable
- Documentation updated for new behavior
- No slop: vague refactors, bulk generated edits, unrelated changes
- CI passes (if applicable)

## Rejected quickly

- Vague drive-by changes
- Changes unrelated to censorship, surveillance, or resilient communication
- Bulk template spam
- Claims without evidence or methodology

## Questions

Open an issue with a `question` label. The maintainer responds.
